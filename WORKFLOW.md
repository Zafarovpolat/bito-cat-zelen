# Рабочий процесс генерации фото - WORKFLOW.md

## Как работаем с v0

### Шаг 1: Ты даёшь мне фотку товара
```
Я беру фото → смотрю её → готов принять промпт
```

### Шаг 2: Ты даёшь промпт (например)
```
"Убрать фон, оставить товар на белом"
или
"Добавить тень снизу, убрать шумы"
или просто фото и я использую дефолтный промпт
```

### Шаг 3: Я запускаю скрипт
```bash
npm run generate -- --input ./your-photo.jpg --output "9001-Z-1-green-1.png" --quality low --prompt "твой промпт"
```

### Шаг 4: Результат готов
```
✓ Фото обработано
✓ Сохранено в: edited/9001-Z-1-green-1.png
```

## Примеры рабочих процессов

### Работа 1: Простая генерация
```
Ты: "Вот фото зелени, сохрани как 9001-Z-1-green-1.png"
Я: запускаю дефолтный скрипт → результат готов
```

### Работа 2: С кастомным промптом
```
Ты: "Вот фото, убери фон и добавь лёгкую тень"
Я: запускаю скрипт с твоим промптом → результат готов
```

### Работа 3: Несколько фото с разными вариантами
```
Ты: 
  Фото 1 → "9001-Z-1-green-1.png, убрать фон"
  Фото 2 → "9001-Z-1-green-2.png, убрать фон"
  Фото 3 → "9010-Z-2-dark-green-1.png, убрать фон и добавить свет"

Я: запускаю по три раза → все результаты готовы
```

## Параметры при необходимости

### Качество
```bash
--quality low      # По умолчанию (быстро, экономно)
--quality high     # Если нужна лучше качество
```

### Размер
```bash
--size 1024x1024   # По умолчанию
--size 512x512     # Меньше/быстрее
--size 2048x2048   # Больше/медленнее (только если нужно)
```

### Полный пример с параметрами
```bash
npm run generate \
  --input ~/photo.jpg \
  --output "9001-Z-1-green-1.png" \
  --prompt "Remove background, add shadow" \
  --quality low \
  --size 1024x1024
```

```
"Remove the background. Place this exact decorative greenery 
product on a pure white background. Do not change any colors, 
texture or shape. Just replace background with clean white."
```

## Логирование

Все операции логируются в `generation-log.jsonl`:
```bash
cat generation-log.jsonl | jq .
```

Каждая запись содержит:
- timestamp
- input файл
- output файл
- модель (gpt-image-2)
- качество (low/high)
- промпт
- успех/ошибка

---

## Быстрые команды

```bash
# Одна фотка
npm run generate -- --input ~/photo.jpg --output "9001-Z-1-green-1.png"

# С промптом
npm run generate -- --input ~/photo.jpg --output "9001-Z-1-green-1.png" --prompt "твой промпт"

# Просмотр логов
cat generation-log.jsonl | tail -10

# После генерации можно сразу загрузить:
npm run sync       # квадрат → Bito → Supabase
```

## Модель и настройки

- **Модель:** gpt-image-2
- **Режим:** edit (редактирование изображений)
- **Качество по умолчанию:** low (быстро и экономно)
- **Размер по умолчанию:** 1024x1024
- **Формат вывода:** PNG (в папке edited/)

---

## Что происходит в фоне

1. Ты даёшь фото + промпт
2. Скрипт читает фото в base64
3. Отправляет на OpenAI API (gpt-image-2)
4. API обрабатывает фото согласно промпту
5. Возвращает новое фото
6. Скрипт сохраняет в edited/ с нужным именем
7. Логирует операцию в generation-log.jsonl
